# Stride Spec

Use this skill when turning an idea into a clear spec before implementation.

## Use when

- The work needs scope definition.
- The user wants questions, boundaries, or a spec.
- The next step should be approval before build work begins.

## Do

- Clarify the goal.
- Identify the main mechanic.
- List assumptions, risks, and non-goals.
- Produce a spec the user can approve.
